# Data Availability

This repository contains everything required to replicate the findings of
*"Outlier Detection Methods Are Not Equally Resilient to Sampling"*, in compliance
with the [PLOS Data Availability policy](https://journals.plos.org/plosone/s/data-availability).

There are two kinds of data:

1. **Derived data generated by this study** — the per-condition resilience results
   (one CSV per experiment) that directly underlie every table and figure. These are
   included in `results/` and are archived with a persistent identifier (Zenodo DOI).
2. **Existing third-party benchmark data** — the 16 public outlier-detection datasets,
   obtained from ADBench. These are not redistributed here (they are git-ignored) but
   are downloaded on demand from their original public source by `download_datasets.py`.

## Obtaining a Zenodo DOI (persistent identifier — required by PLOS)

GitHub alone does not provide long-term preservation, so PLOS requires a DOI. Link the
repository to Zenodo once; every GitHub release then mints a citable DOI:

1. Sign in at <https://zenodo.org> with your GitHub account.
2. In Zenodo → **GitHub** settings, toggle the `LaureBerti/OD_resilience` repository **On**.
3. On GitHub, create a release (e.g. tag `v2.0.0`). Zenodo automatically archives it and
   mints a DOI of the form `10.5281/zenodo.XXXXXXX`.
4. Put that DOI in the Data Availability Statement above (and in `CITATION.cff`).

> A placeholder DOI is acceptable at first submission; finalise it before publication.
